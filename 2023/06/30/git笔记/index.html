文件版本保存缺点

  • 操作麻烦-每次需要复制粘贴重命名
  • 明明不规范-无法通过文件名知道具体做了那些修改
  • 容易丢失-如果硬盘故障或不小心删除，文件很容易丢失
  • 写作困难-需要手动合并每个人对项目文件的修改，合并时极易出错

版本控制软件

1.概念，记录文件变化，以便将来查阅特定版本修订情况的系统
2.通俗理解：把手工管理文件版本的方式，改为由软件管理文件的版本，这个负责管理文件版本的软件，叫做版本控制软件。

优点

  • 操作简便-只需记几组简单的终端命令
  • 易于对比
  • 易于回溯
  • 不易丢失
  • 协作方便

分类

本地版本控制系统（单机运行）、集中化版本控制系统（多人开发、性能差，用户体验不好）、分布式版本控制系统（多人开发、性能优秀、用户体验好）

集中化版本控制系统（SVN）

特点：基于服务器、客户端的运行模式

  1. 服务器保存文件的所有更新记录
  2. 客户端只保留最新的文件版本
    优点：联网运行、支持多人协作
    缺点：
    1.不支持离线提交版本更新
    2.中心服务器崩溃后，所有人无法正常工作
    3.版本数据库故障后，所有历史更新记录会丢失

分布式版本控制系统（git）

特点：基于服务器、客户端的运行模式

  1. 服务器保存文件的所有更新版本
  2. 客户端是服务器的完整备份，并不是只保留文件的最新版本

优点：
1.联网运行，支持多人协作开发
2.客户端断网后支持离线本地提交版本更新
3.服务器故障或损坏后，可使用任何一个客户端的备份进行恢复。

git的基本操作

1.获取git仓库的两种方式：

  • 将尚未进行版本控制的本地目录转换为git仓库
  • 从其他服务器克隆一个已存在的git仓库

2.在现有目录中初始化仓库
文件夹进入终端命令行，复制命令上传文件
3.查看文件工作区状态命令 git status，精简输出命令 git status -s
4.跟踪新文件（添加到暂存区）：git add filename
5.提交更新git commit -m “commit message”
6.撤销修改git checkout – index.html，原理：用git仓库中的旧版本覆盖修改的文件，不可撤销，危险性较高，慎重操作
7.取消暂存的文件：git reset HEAD filename
8.跳过暂存区，git commit -a -m “commit message”，执行这个命令不再需要执行 git add 命令
9.移除文件：从git仓库和工作区同时移除文件 git rm -f filename
只从git仓库中移除文件 git rm –cached filename
10.忽略文件：创建.gitignore配置文件，列出要忽略的文件的匹配模式

  • 以#开头为注释
  • 以/结尾为目录
  • 以/开头防止递归
  • 以！开头表示取反
  • 可以使用glob模式（简化的正则表达式）进行文件和文件夹的匹配
    1. * 匹配零个或多个任意字符
    2. [abc] 匹配任何一个列在方括号中的字符（匹配一个a/一个b/一个c）
    3. ？ 只匹配一个任意字符
    4. 0-9 匹配两个字符范围内的都可以匹配到
    5. a/**/z 表示匹配任意中间目录 a/z/ ,a/sddsfd/s/z/ ,a/b/z
