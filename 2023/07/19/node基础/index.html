node基础

Posted on 2023-07-19 Edited on 2023-07-30 In Web , JS

node.js是一种基于Chrome V8引擎的 JavaScript 运行环境。浏览器是js的前端运行环境，node.js是js的后端运行环境，node.js无法调用BOM、DOM等WebAPI

1. node的核心理念

node的核心理念是事件驱动编程。

http请求就是要处理的事件，http.creatServer方法将函数作为一个参数，每当发生http请求，再回调用http.creatServer（）中的函数对事件进行响应。

2. node的运行环境

3. node的作用

Node.js 作为一个 JavaScript 的运行环境，仅仅提供了基础的功能和 API。

  1. 基于express框架，可以快速构建 Web 应用
  2. 基于 Electron 框架，可以构建跨平台的桌面应用
  3. 基于 restify 框架，可以快速构建 API 接口项目
  4. 读写和操作数据库、创建实用的命令行工具辅助前端开发、etc…

4. node的内置API

内置API模块使用方法：使用require方法导入
let fs = require('fs')

（1）fs：文件系统

readFile(path[,option],callback);writeFile(path,data,[,option],callback)
Tips:
[,option]可选参数为文件读写的编码格式，一般为’utf-8’
读文件回调函数两个参数为err和result；写文件回调函数参数为err。

（2）path：路径的拼接与获取文件名

  • path.join()将多个路径片段拼接成一个完整的路径字符串,返回字符串;路径拼接尽量不要使用+
  • path.basename(path[,ext])用来从路径字符串中，将文件名解析出来
    path必选参数，表示一个路径的字符串;ext可选参数，表示文件扩展名
    可选参数不填，返回带后缀的文件名，填文件后缀名，则只返回文件名
  • path.extname(path) 方法，可以获取路径中的扩展名部分

（3）http：创建 web 服务器。

创建 web 服务器的基本步骤：
① 导入 http 模块
const http = require('http')
② 创建 web 服务器实例
const server = http.createServer();
③ 为服务器实例绑定 request 事件，监听客户端的请求
回调函数req，res参数为对象，req为请求对象，拥有url和method属性。

④ 启动服务器

Tips:不是每次访问页面都会输入文件名，优化路径，使得访问跟目录时，仍旧返回主页内容。文件的读取封装到单独的函数中处理，见第6节静态服务资源。

5.node模块化

模块化是指解决一个复杂问题时，自顶向下将系统划分为若干个子模块的过程，即每个单独的功能封装在一个模块中，从而提高代码的复用性。

（1）模块化的优点

  • 提高了代码的复用性
  • 提高了代码的可维护性
  • 可以实现按需加载

Node.js模块化使用步骤：1.require方法导入 2.利用module.exports对象向模块外暴露成员

Tips:node提供了exports对象来简化module.exports，实现原理即指向同一个地址，但如果exports或者module.exports输出为对象时，会单独重新开辟空间，此时指向不为同一地址，因此在使用时，不要在一个模块中同时使用

（2）npm包

利用别人封装好的模块来快捷完成需求

安装与使用

npm install 包名

安装指定版本：包名后加艾特包名，无空格

npm install 包名@x.xx.x

Tips：install 可以缩写为i

安装成功后，项目文件夹会多一个node_modules的文件夹和package-lock.json（记录每个包的下载信息）的配置文件

由于node_modules文件夹一般比较大，多人协作时，共享项目提出node_modules，在项目根目录创建一个package.json的配置文件，记录项目使用哪些包即可，.gitignore需要忽略node_modules文件夹,快速创建package.json文件（一般会自动创建）：

npm init -y

package.json文件中的depenencies节点：记录npm install 命令安装了哪些包,当提出node_modules文件夹后，如下命令可以快速安装项目所用的包（根据depenencies）：

npm install(i)

devDepenencies：项目开发阶段使用的包，上线后不会用到，安装命令

npm i 包名 -D

npm作为国外网站，下载速度过慢，安装nrm包快速切换镜像资源

命令：

工具性质的包可以安装全局包，命令：

npm i 包名 -g

（3） 自定义npm包

创建包的结构：

1.包必须以单独的文件夹存在

2.包的根目录下必须有package.json这个包管理配置文件

3.package.json中必须包含name（包名）, version（版本）,main（入口）这三个属性。

1）初始化包的基本结构

建立date-tools文件夹，作为包的根目录，并新建三个文件：

index.js:包的入口文件；package.json：包管理配置文件；README.md包说明文档

  • 初始化package.json文件

其中name、version、main为必填项

  • 在index.js封装方法（以格式化时间功能为例）

  • 编写包的说明文档

将包的使用说明写入README.md文档,内容一般包括如下内容：安装方式、导入方式、格式化事件、转义HTML中的特殊标签，、还原HTML中的特殊字符、开源协议

2）发布包

注册npm账号，在终端使用命令登录

npm login

将终端切换至包的根目录下，运行发布命令：

npm pulish

删除已发布包命令：

npm unpulish

（4）一个模块包含多个方法处理办法

  • 创建一个src文件夹，将多个方法放在src文件夹中

  • 在index.js文件中导入多个方法。

  • 在需要使用包的js中利用require方法调用即可使用。

6.路由的概念

路由是指应用程序如何响应客户端发出的请求内容。广义来讲，路由就是映射关系。对于web客户端/服务器来说，客户端在URL中指明它想要的内容。服务器端根据客户端的请求选择相应的处理逻辑的方式就是路由。

7.静态服务资源

在有了路由之后，可以配置一些html和img的静态资源（不会因刷新而更新）

8.Express

使用node.js可以实现路由与配置静态资源，但基本上都是在重复Apache或IIS可以自动完成的工作，这使Node看起来没有竞争力，但是Express框架为Node.js带来了简便的应用WEB扩展程序，省去大量编写基础设施类代码的时间。
